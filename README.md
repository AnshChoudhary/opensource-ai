# Y Decombinator

**Open-source alternatives to Y Combinator AI startups. Stop paying for wrappers.**

## The Initiative

Y Decombinator is a critical commentary on the VC-backed "wrapper" startup culture. We believe that many Y Combinator startups are selling UIs for public AI models—features, not companies. These services don't need to be venture-backed companies charging $30-200/month subscriptions.

### Our Mission

To democratize AI utility and return it to the people. We believe the logic of the modern web belongs in the open, not behind subscription fees. By providing free, open-source alternatives, we:

- **Challenge the "Wrapper" Problem**: Expose how many startups are just UIs around public AI models
- **Promote Data Sovereignty**: Enable users to host services locally, keeping their data private
- **Prevent "Feature-as-a-Service"**: Turn specialized AI tools into public utilities, not corporate paywalls
- **Lower Barriers**: Make AI tools accessible to students, researchers, and small creators

## The Problem

### The "Wrapper" Problem

Many startups provide a specific UI for a general AI task (e.g., "AI for legal PDFs" or "AI for SQL queries"). These are features, not companies. A single well-maintained open-source tool can offer the same utility without the monthly B2B subscription fee.

### Low Technical Moats

Because many startups rely on the same third-party APIs (OpenAI, Anthropic), they don't own the underlying "engine." If the tech is easy to replicate, it should be a public good rather than a venture-backed company.

### Data Sovereignty

When you use a B2B startup, your data is their product. Open-sourcing these services allows users to host them locally (or on their own cloud), ensuring sensitive information never leaves their sight.

### Economic Fairness

By providing free, open-source alternatives, we lower the barrier to entry for students, researchers, and small creators who are currently priced out by the "SaaS-ification" of every basic AI task.

## How It Works

### B2B Stack (What We're Replacing)
```
User → Proprietary Layer → Subscription Fee → AI Model
```
Your data goes through a third-party server. You pay monthly regardless of usage.

### Open-Source Stack (Our Alternative)
```
User → Open-Source Interface → Your API Key → AI Model
```
Direct connection. You control the data. Pay only for API usage (or run locally for free).

## Bring Your Own Key (BYOK)

We're honest about costs. While our services are free and open-source, you'll need to provide your own API keys (OpenAI, Anthropic, etc.) for the underlying models. This means:

- You pay for what you use, not for the privilege of using it
- No monthly subscription fees or vendor lock-in
- Complete control over your API usage and costs
- Option to run local models for zero API costs
